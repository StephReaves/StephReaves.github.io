Classes and more classes

Classes are a big concept in the sense that they give you a lot of control and scalability in your work. Everything in Ruby is an object and classes are not an exception. I've started thinking of classes as categories. So for example, you might want to create an Dog class. Each dog can have specfic characteristics such as: breed, name, weight, and gender. Within the class there are instance variables, class variables, and local variables. As you form your class you will want to start it (initialize it) with instance variables. These instance variables contant an @ in front of the variable name. So if you wanted to have all of the charactersitics of a dog discussed earlier be variables then you would include the following: @breed, @name, @weight, @gender. The @ sign in front of the variable name notes that these are instance variables.

If the variables of name, weight, and gender, were included in a specific method, then these variables would be local variables and only accessible to the specific method in which it is contained. As you can imagine when you are writing a class, it is preferable to have some variables that can traverse across all methods within the class.

The final variable type is the class variable. The class variable is accessible to all members of a class. The scope of the class variable is wider than the instance variable, which has a wider scope than the local variable. The class variable is signified by @@. So if we follow the Dog example we can create a class variable called number_of_eyes and this would be coded as @@number_of_eyes.

One final thought, is that classes can have parents and children classes. It's kind of like taxonomy. If I wanted to write software for an pet store, I could create a superclass of Animal of which Dog would be a subclass. Dog would inheirit all behaviors (methods) of its parent class.
